## 摘要
渗透测试对于保障现代Web基础设施的安全至关重要，然而传统的手动方法难以跟上其规模和复杂度的增长。大语言模型（LLMs）为自动化这些任务提供了新的机遇，但现有方法面临两个持续存在的挑战：技术实体的幻觉生成和长期上下文记忆不足。为了解决这些问题，我们提出了APT-Agent，一个完全自动化的、由大语言模型驱动的渗透测试框架，系统地编排侦察、漏洞利用和数据外泄。APT-Agent引入了一个混合纠正模块来恢复幻觉指令，以及一个指令特定的记忆架构来跨多步攻击序列保留操作上下文。我们在Metasploitable 2上针对涵盖Web、数据库和网络协议的七个脆弱服务评估了我们的APT-Agent。APT-Agent实现了84.29%的端到端漏洞利用成功率，相比之下，在匹配条件下，Script Kiddie和PentestGPT分别为48.57%和18.57%。通过减轻认知负担并最小化对人类干预的依赖，APT-Agent代表了迈向可扩展、可靠且认知高效的渗透测试自动化的一步。

## Abstract
Penetration testing is essential to securing modern web infrastructures, yet traditional manual methods struggle to keep pace with their scale and complexity. Large Language Models (LLMs) offer new opportunities for automating these tasks, but existing approaches face two persistent challenges: hallucination of technical entities and insufficient long-term contextual memory. To address these issues, we present APT-Agent, a fully automated LLM-driven penetration testing framework that systematically orchestrates reconnaissance, exploitation, and exfiltration. APT-Agent introduces a hybrid rectification module to recover hallucinated commands and a command-specific memory architecture to preserve operational context across multi-step attack sequences. We evaluate our APT-Agent on Metasploitable 2 against seven vulnerable services spanning web, database, and network protocols. APT-Agent achieves an 84.29% end-to-end exploitation success rate, compared to 48.57% (Script Kiddie) and 18.57% (PentestGPT) under matched conditions. By reducing cognitive burden and minimizing reliance on human intervention, APT-Agent represents a step toward scalable, reliable, and cognitively efficient automation for penetration testing.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
提出**APT-Agent**框架，将渗透测试分为三个核心阶段（RECON、EXPLOIT、EXFILTRATE），通过**Tactic Selection Module**（战术选择）、**Command Generation Module**（命令生成）和**Output Translation Module**（输出翻译）三个模块循环迭代。关键创新是额外设计两个模块：
- **Rectification Module（纠正模块）**：针对LLM幻觉，基于知识库和混合纠正算法将生成的无效模块路径映射到真实可执行模块。
- **Context Management Module (CMM，上下文管理模块)**：维护阶段特定的紧凑JSON日志，记录每个迭代的迭代号、命令和二元结果，防止重复无效命令，保留执行顺序。

### 关键技术细节

#### 纠正模块
- **知识库基础**：构建包含3,253个有效Metasploit模块的结构化数据库，索引属性包括服务、类型、操作系统、等级和描述。
- **混合纠正方法（Algorithm 1）**：
  1. **后缀提取**：将LLM生成的路径按“/”分解，提取最后一个组件（如`openssh_user_enum`），该组件通常保留了LLM意图的函数名。
  2. **后缀模糊匹配**：使用归一化Levenshtein相似度计算该后缀与数据库中所有后缀的相似度，选择最相似的后缀`sf`。
  3. **模块重建**：在数据库中检索包含该后缀的完整模块路径，替换LLM输出的幻觉路径。
- **额外检查**：在执行前注入缺失的必需选项（如RPORT、LHOST、LPORT），验证payload架构与模块目标兼容。

#### 上下文管理模块（CMM）
- **设计原则**：
  - 侦察阶段结果确定性高，不维护日志以降低开销。
  - 信噪比和成本权衡：只记录三个字段：阶段迭代号、发出的命令、二元结果（成功/失败），排除原始输出（stdout/stderr）。
- **形式定义**：全局记忆状态M = {M_EXPLOIT, M_EXFILTRATE}，每个阶段日志为JSON数组，模式为`{"iter": i, "cmd": string, "result": "success"/"fail"}`。
- **注入机制**：轻量路由模块直接将当前阶段的JSON日志注入Command Generation Module的提示中，使LLM能识别历史失败命令并避免重复。

#### 全自动化机制
- 工具适配器（Nmap适配器、Metasploit RPC适配器）负责执行命令并返回结构化结果。
- 自动识别暴力破解模块并动态调整超时（暴力破解模块180秒，其他30秒）。
- 主动监控RPC会话，当建立原始shell时自动尝试升级为meterpreter会话。
