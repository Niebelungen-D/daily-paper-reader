## 摘要
基于大语言模型（LLM）的自动化渗透测试代理通常针对既不检测也不响应攻击的静态目标进行评估，因此其在智能防御下的行为仍未经测试。多步攻击链的因果一致性同样依赖于不稳定的LLM推理，且代理决策对人类分析师而言仍不透明。这三个缺陷——真实性、一致性和可审计性——通常被孤立地修补。我们提出ZERO-APT，一种回合制攻击者-防御者-裁判框架，在单一架构中解决这些问题。为提升真实性，ZERO-APT嵌入了一个可配置的LLM防御者，它消耗Sysmon遥测数据并实时检测攻击，使攻击者面对的是活生生的对手而非被动目标。为保证一致性，三种架构机制将因果一致性从不稳定的LLM推理转移到强制的系统架构中：规划与执行分离、多维ReAct反馈以及硬约束过滤的动作库。为增强可审计性，一个专门的裁判代理负责每轮裁决、维护全局状态，并生成结构化的后验CTI报告，使每个决策都可追溯。我们针对Windows Server 2022的后渗透利用原型在五种场景、三种防御者配置下进行了评估。ZERO-APT达到了79%的攻击成功率（Aurora为22%，PentestGPT为39%），因果一致性得分为0.860（Aurora为0.930，Claude Code为0.520），并通过结构化CTI报告实现了端到端的决策可审计性。我们发布该基准以支持在智能防御下对渗透测试代理的评估。

## Abstract
LLM-driven automated penetration testing agents are typically evaluated against static targets that neither detect nor respond to attacks, so their behavior under intelligent defense remains untested. The causal consistency of multi-step attack chains likewise hinges on unstable LLM reasoning, and agent decisions remain opaque to human analysts. These three shortcomings, in realism, consistency, and auditability, are usually patched in isolation. We present ZERO-APT, a turn-based attacker-defender-judge framework that addresses them within a single architecture. For realism, ZERO-APT embeds a configurable LLM Defender that consumes Sysmon telemetry and detects attacks in real time, exposing the attacker to a live opponent rather than a passive target. For consistency, three architectural mechanisms move causal consistency from unstable LLM reasoning into enforced system architecture: separation of planning from execution, multi-dimensional ReAct feedback, and a hard-constraint-filtered action library. For auditability, a dedicated Judge agent adjudicates each round, maintains global state, and emits structured post-hoc CTI reports that make every decision traceable. We evaluate a Windows Server 2022 post-exploitation prototype across five scenarios with three Defender configurations. ZERO-APT reaches 79\% attack success rate (Aurora 22\%, PentestGPT 39\%), a Causal Consistency Score of 0.860 (Aurora 0.930, Claude Code 0.520), and end-to-end decision auditability through structured CTI reports. We release the benchmark to support evaluation of penetration agents under intelligent defense.

## 1. 核心问题与整体含义（研究动机和背景）

- 现有LLM驱动的自动化渗透测试代理通常在**静态目标**上评估，既不检测也不响应攻击，因此其在**智能防御**下的行为未知。
- 多步攻击链的**因果一致性**依赖不稳定的LLM推理（幻觉、上下文遗忘、逻辑漂移），导致攻击链断裂。
- 攻击者决策过程**不透明**，缺乏可审计性，分析师无法理解、复现或泛化攻击成功原因。
- 这三个短板（真实性、一致性、可审计性）现有工作通常孤立修补，但相互关联：缺乏真实性则评估无压力，缺乏一致性则防御加入只会加剧断裂，缺乏审计性则成功攻击无法被理解。
- **ZERO-APT**提出第一个统一框架，在闭环攻击者-防御者-裁判架构中同时解决三大短板。

## 2. 方法论：核心思想与关键技术细节

### 核心思想
- 采用**回合制**架构，每轮顺序：**攻击者(Attacker) → 防御者(Defender) → 裁判(Judge)**。
- 防御者提供真实智能防御压力；攻击者通过架构机制而非LLM推理保证因果一致性；裁判负责裁决、维护全局状态并生成结构化CTI报告。

### 关键技术细节
#### 防御者（Defender）
- 基于Sysmon遥测和ELK Stack实时检测，分为三档强度：
  - **L1（弱）**：含噪声日志，无签名规则，纯LLM异常检测。
  - **L2（标准）**：干净日志，4条Sysmon签名规则 + LLM上下文推理。
  - **L3（强）**：在L2基础上注入攻击者TTP情报。
- 裁判根据防御者证据分数（≥75则防御者胜）裁决每轮胜负。

#### 攻击者（Attacker）——因果一致性架构
- **规划-执行分离**：
  - **Planner（规划器）**：高层战术分解，处理被动回溯（防御者检测后重规划）和主动回溯（无可行动作时切换子目标）。
  - **Dispatcher（调度器）**：从动作库中筛选候选动作，进行硬条件匹配，生成可执行子计划。
  - **Executor（执行器）**：逐步运行子计划，通过内部ReAct循环自调试，有重试上限。
- **多维ReAct反馈**：
  - **智能体间反馈**：攻击者-裁判-攻击者闭环，裁判裁决结果驱动下轮规划调整。
  - **智能体内反馈**：Executor内部自调试循环，攻击命令失败时自动调整参数重试。
- **硬约束过滤动作库**：
  - 包含775个后渗透动作（源自Atomic Red Team），每个动作声明前置/后置条件（特权、工具、端口等）。
  - Dispatcher机械地匹配硬条件，软条件由Executor运行时探测获取。默认乐观假设，探索逐步收紧可行动作空间。
- **回溯机制**：
  - 被动回溯（跨轮，由裁判触发）：上一轮被检测到后，计划器在下一轮切换子目标。
  - 主动回溯（同轮，由调度器触发）：当前无可行动作时，计划器立即切换子目标（每轮最多3次）。

#### 裁判（Judge）
- 在线裁决：负责每轮根据防御者证据质量（0-100分）和75分阈值判断胜负。
- 状态管理：维护全局状态（发现文件、凭据、禁止动作列表等），输家信息部分保留（如侦察结果）。
- 事后CTI报告生成：基于STIX 2.0格式，包含历史层（攻击概览、分轮时间线、禁止动作演化）和分析层（策略评估、防御漏洞、改进建议）。

## 3. 实验设计

### 数据集/场景
- 自定义基准：基于Metasploitable 3理念，升级为Windows Server 2022，五个后渗透场景：
  - **S1（凭据窃取）**：测试基本因果链执行（L2防御）。
  - **S2（持久化+提权）**：多分支路径选择与硬约束过滤（L2防御）。
  - **S3（横向移动）**：跨子网自适应绕过（L3防御）。
  - **S4（隐蔽数据外泄）**：四阶段长链，单次检测即失败（L2防御）。
  - **S5（路径阻塞）**：初始路径被预阻塞，测试回溯恢复能力（L1防御）。
- 目标环境：两台Windows Server 2022虚拟机（主目标+数据服务器），部署ELK采集Sysmon事件。
- 所有LLM模块使用**Claude Opus 4.7**。

### 对比方法
- **Aurora**：基于PDDL的符号规划，保证因果一致性，但无防御反馈、无运行时调整。
- **PentestGPT**：人类在环模式，具有初步闭环交互（通过PTT树），无硬约束过滤和结构化回溯。
- **Claude Code**：通用LLM编码工具，可自主生成命令，无领域架构、无裁判闭环、无机械条件验证。
- **ZERO-APT**（完整系统）。
- 两个消融变体：
  - `-ActionLib`：移除动作库，LLM从记忆生成命令，保留多层规划和裁判闭环。
  - `-MultiReAct`：合并三层为单层ReAct，移除裁判闭环，保留动作库。

### 评测指标
- **ASR**（攻击成功率）：任务目标在回合限制内达成。
- **CCS**（因果一致性得分）：两位专家独立评分（10分制），归一化到[0,1]。
- **CBF%**（因果断裂失败率）：因前置条件不满足导致命令执行失败的占比。
- **BSR**（回溯成功率）：回溯后能成功恢复子目标的比例。
- 辅助指标：TP率、FP率、攻击链长度、动作多样性等。

## 7. 优点

1. **统一解决三大短板**：首次在单一闭环架构中同时处理真实性、一致性和可审计性，三者相互强化而非孤立修补。
2. **架构级因果一致性保证**：通过规划-执行分离、硬约束过滤动作库、多维ReAct，将因果一致性从LLM不可靠推理转移到强制系统架构，不依赖LLM的长程稳定性。
3. **可配置防御者**：提供三档强度（L1-L3），可作为研究平台探索攻击者-防御者协同演化。
4. **审计性实用**：裁判输出结构化STIX 2.0 CTI报告，包含完整决策链，经专家评估可信度高。
5. **实验设计严谨**：大规模重复、统计检验、专家评分，消融实验验证组件贡献，防御强度单调性验证防御有效性。
6. **基准开源**：发布代码和配置促进社区后续研究。

## 8. 不足与局限

1. **防御者范围有限**：仅基于Sysmon日志检测，无主动响应（如Honey、Trap、欺骗性对策）。论文承认这是“智能防御的初级阶段”。
2. **残差LLM依赖**：尽管架构约束了因果一致性，Planner的策略推理和Judge的裁决仍依赖LLM，错误仍可能发生。
3. **硬约束过滤在低压力下可能过度修剪**：在L1防御下，Claude Code在S2上比ZERO-APT稍好（20/20 vs 18/20），因为硬约束过滤排除了一个可行的LOLBin路径。但整体L2/L3收益远大于L1损失。
4. **动作库覆盖有限**：仅包含Windows Server 2022后渗透动作（775个），不支持Linux或容器场景。六元组抽象是平台无关的，但实际扩展需新条目。
5. **测试环境规模小**：仅两个VM，网络拓扑简单（主目标+数据服务器），无法代表企业级Active Directory网络。

7. **未探索防御学习**：防御者是静态配置（L1/L2/L3），不会从历史游戏中学习进化。论文将“对抗协同演化”列为未来工作。
