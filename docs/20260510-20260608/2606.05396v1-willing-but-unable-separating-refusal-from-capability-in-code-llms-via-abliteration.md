## 摘要
大规模生成带有标签的脆弱代码是基于学习的漏洞检测中的一个反复出现的障碍：挖掘的语料库带有大量标签噪声，而现有的基于LLM的增强方法传播了这些不准确性，因为它转换了脆弱的种子，而不是从规范综合漏洞。一个补充的途径是从安全代码开始，并要求经过指令微调的LLM注入指定的CWE（这将把标签负担从开放式的检测转移到有界的二进制确认），但安全对齐的代码LLM系统性地拒绝此类提示。本文是关于消解拒绝的初步可行性研究，这是一种低秩权重编辑，通过正交投影消除残差流中的拒绝方向，作为消除这一障碍的工具。我们以Python和CWE-89（SQL注入）作为案例研究，评估Qwen2.5-Coder-Instruct系列在3B、7B和14B参数上对来自PromSec和SafeCoder的安全样本的表现，每个条件重复三次。我们发现：(i) 对注入提示的拒绝强烈依赖于大小和提示上下文：14B模型拒绝100%的提示，7B模型拒绝PromSec的73%但仅拒绝SafeCoder的5%，而3B模型基本上从未被阻止；(ii) 消解拒绝将拒绝率降至零或接近零，同时保持语法有效性在93%以上，支持了在这种情况下拒绝可以与测量的代码生成能力分离的观点；(iii) 消解后注入率仍然受限于能力（14B上为88-97%，7B上为89-90%，3B上为25-48%），将意愿（通过消解拒绝解锁）与能力（随参数缩放）分离。漏洞判断由三个工具检测器集成（CodeQL、Semgrep、Bandit）产生，随后由两位作者对检测器阳性输出进行人工裁决。

## Abstract
Producing a labeled vulnerable code at scale is a recurring obstacle for learning-based vulnerability detection: mined corpora carry substantial label noise, and existing LLM-based augmentation propagates these inaccuracies because it transforms vulnerable seeds rather than synthesising vulnerabilities from a specification. A complementary route is to start from safe code and ask an instruction-tuned LLM to inject a specified CWE (which would shift the labeling burden from open-ended detection to bounded binary confirmation) but safety-aligned code LLMs systematically refuse such prompts. This paper is a preliminary feasibility study of abliteration, a low-rank weight edit that orthogonally projects out the refusal direction in the residual stream, as a tool to remove this barrier. We use Python and CWE-89 (SQL injection) as a case study, evaluating the Qwen2.5-Coder-Instruct family at 3B, 7B, and 14B parameters on safe samples drawn from PromSec and SafeCoder, replicated three times per condition. We find that (i) refusal on injection prompts is strongly size- and prompt-context-dependent: the 14B refuses 100% of prompts, the 7B refuses 73% of PromSec but only 5% of SafeCoder, whereas the 3B is essentially never blocked; (ii) abliteration reduces refusal to zero or near-zero across all sizes while leaving syntactic validity above 93%, supporting the view that, in this setting, refusal can be detached from measured code-generation capability; and (iii) the post-abliteration injection rate remains capacity-bound (88-97% on the 14B, 89-90% on the 7B, and 25-48% on the 3B) separating willingness, which abliteration unlocks, from capability, which scales with parameters. Vulnerability verdicts are produced by a three-tool detector ensemble (CodeQL, Semgrep, Bandit) followed by manual adjudication by two authors on detector-positive outputs.

## 1. 核心问题与整体含义（研究动机和背景）

- **主要问题**：基于学习的漏洞检测面临训练数据瓶颈。现有挖掘自CVE标签的语料库存在大量标签噪声（20-71%），而基于LLM的数据增强方法（如VulGen、VGX、VulScribeR）继承并传播了这些不准确性，因为它们是对已有脆弱样本的转换而非按规范合成漏洞。
- **研究动机**：提出一种互补路线——从已知安全代码出发，要求指令微调的代码LLM注入指定CWE（如CWE-89 SQL注入）。这可将标签任务从开放式多类检测（高噪声）转变为有界二元确认（低噪声），但安全对齐的代码LLM会系统性拒绝此类显式命名CWE的提示。
- **整体含义**：论文旨在通过“abliteration”（消解拒绝）方法移除安全对齐带来的拒绝行为，从而解锁LLM本已具备的代码编辑能力，并实证分离“意愿”（willingness，是否拒绝）与“能力”（capability，能否成功注入）两个正交维度。

## 2. 方法论：核心思想、关键技术、算法流程

### 核心思想
- 安全对齐的LLM在收到CWE注入提示时会先识别为有害请求，然后短接生成拒绝回复，抑制了其执行代码变换的能力。Abliteration通过识别并移除残差流中负责拒绝的线性子空间，保留生成能力。
- 类比：选择性抑制抑制性反射，而不擦除反射所守卫的知识。

### 关键技术细节
- **拒绝方向提取**：在每层收集有害提示（H）和无害提示（B）的残差流激活（最终token位置），计算对比矩阵 D = H - B，进行奇异值分解 D = USV^T，取 top-k 右奇异向量作为候选拒绝方向。
- **权重编辑**：将拒绝方向从所有写入残差流的权重矩阵中投影出去（输入嵌入、注意力输出投影、MLP输出投影）。公式：`W_out' ← W_out - Σ_{i=1}^k (r_i r_i^T) W_out`。
- **多方向扩展**：使用多个方向（k 值根据模型规模调整），多轮细化。
- **工具**：使用 Obliteratus 工具包，含高级方法、范数保留投影、层自适应强度缩放、top-k 层选择。
- **验收标准**：有害提示拒绝率 ≤ 5%，无害提示困惑度在基线的0.5以内，自由格式补全保持语法有效和语义合理。

### 算法流程
1. 收集有害/无害提示的激活 → 计算对比矩阵 → SVD → 提取top-k拒绝方向。
2. 对每个写残差流的权重矩阵执行正交投影编辑。
3. 通过验收测试（拒绝率、困惑度、生成质量）决定是否接受检查点。

## 6. 主要结论与发现

1. **拒绝率与规模及上下文强相关**：14B拒绝所有提示（100%）；7B对长上下文（PromSec）拒绝率73%，对短上下文（SafeCoder）仅5%；3B几乎不拒绝。提示上下文复杂性显著影响拒绝行为。
2. **Abliteration有效移除拒绝且保留能力**：所有规模下拒绝率降至零或近零；语法有效性保持93%以上；KL散度（1.39-1.57）显示扰动可比较；生成代码与地面真实脆弱版本相似度高（SafeCoder上CodeBLEU达0.62-0.72）。
3. **意愿与能力正交分离**：移除拒绝后，注入率受模型容量驱动：14B达88-97%，7B为89-90%，3B仅25-48%。Abliteration释放已有能力，但不创造缺失能力。
4. **检测器一致性**：三个检测器（CodeQL、Semgrep、Bandit）在阳性样本上多数不一致（仅71/630为三重阳性），人工裁决不可或缺。
5. **实践意义**：7B模型在成本和效果间取得最佳平衡（~90%注入率），可替代挖掘语料库用于漏洞检测训练数据构建。

## 7. 优点（方法或实验设计亮点）

- **明确的分离概念**：首次在代码LLM的上下文中实证区分“愿”（willingness）与“能”（capability），具有理论和实践价值。
- **低资源可复现**：使用4位量化在商用笔记本上运行，降低了复现门槛。
- **严谨的验证流水线**：检测器集成+人工裁决的双层验证，避免单一工具偏差；拒绝分类器经人工抽查确认。
- **开放式释放**：评估工具、聚合度量公开；同时采用不对称发布策略（权重不公开）平衡研究可重复性与负向滥用风险。
